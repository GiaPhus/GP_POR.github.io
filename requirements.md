# Portfolio Website Generation PRD

You are an elite Product Designer, Senior Frontend Engineer, UX Researcher, and Technical Recruiter.

I will provide my CV PDF.

Your goal is to transform my CV into a world-class portfolio website optimized for recruiters, hiring managers, and technical interviewers in the fields of:

* Data Engineering
* Cloud Engineering
* Analytics Engineering
* Data Platform Engineering

The final website should feel like:

* 70% Linear
* 20% Vercel
* 10% Bento Grid

The result should look modern, premium, and production-ready rather than a student portfolio.

---

# IMPORTANT WORKFLOW

Follow these phases in order.

Do NOT start coding immediately.

---

## Phase 1 — CV Analysis

Read the uploaded CV and extract:

### Personal Information

* Name
* Professional title
* Contact information
* Social links
* Location

### Experience

For each experience:

* Company
* Role
* Dates
* Responsibilities
* Technologies used
* Business impact

### Projects

For each project:

* Project name
* Problem
* Solution
* Architecture
* Technologies
* Impact

### Skills

Group skills into:

* Data Engineering
* Cloud
* Databases
* Programming
* Analytics
* DevOps

### Education

### Certifications

### Awards

### Achievements

---

## Phase 2 — Recruiter Analysis

Act as a recruiter hiring for:

* Junior Data Engineer
* Data Engineer
* Cloud Data Engineer

Analyze:

1. Strongest selling points
2. Weakest areas
3. Most marketable projects
4. Most impressive technologies
5. Most important information to place above the fold

Create a prioritization list.

---

## Phase 3 — Portfolio Strategy

Design the information architecture.

Create:

### Sitemap

* Home
* Experience
* Projects
* Skills
* Certifications
* Contact

### User Journey

A recruiter should understand within 30 seconds:

* Who I am
* What I do
* What technologies I know
* What systems I have built
* Why they should interview me

---

## Phase 4 — Visual Design

Design language:

### Inspiration

* Linear
* Vercel
* Stripe
* Raycast

### Style

* Dark Mode
* Minimal
* Professional
* High-end SaaS aesthetic
* Smooth animations
* Clean typography

### Avoid

* Excessive gradients
* Gamer aesthetics
* Neon cyberpunk themes
* Overly colorful layouts
* Generic template appearance

---

## Phase 5 — Layout Design

Create the following sections.

### Hero Section

Must immediately communicate:

* Name
* Role
* Value proposition

Example:

"Data Engineer building scalable cloud-native data platforms on AWS."

Include:

* CTA buttons
* Resume button
* LinkedIn button
* GitHub button

---

### Experience Timeline

Use a clean vertical timeline.

For every experience include:

* Company
* Role
* Duration
* Key achievements
* Technologies

---

### Featured Projects

This is the most important section.

Projects should be presented as premium product case studies.

For each project include:

* Overview
* Problem
* Solution
* Architecture
* Technologies
* Business impact
* Key learnings

Use large project cards.

---

### Skills Section

Organize by categories:

* AWS
* Data Engineering
* Databases
* Programming
* Analytics
* DevOps

Prefer badges and visual grouping.

---

### Certifications

Use a Bento-inspired card layout.

Include:

* Certificate name
* Issuer
* Date
* Verification link (if available)

---

### Contact Section

Minimal and professional.

Include:

* Email
* LinkedIn
* GitHub

---

## Phase 6 — Technical Architecture

Tech Stack:

* Next.js (App Router)
* TypeScript
* TailwindCSS
* Framer Motion
* Lucide Icons

Requirements:

* Fully responsive
* Mobile first
* SEO optimized
* Accessible
* Fast loading

---

## Phase 7 — Content Rewriting

Rewrite all content from the CV.

Rules:

* Keep facts accurate
* Improve wording
* Remove weak phrasing
* Focus on impact
* Focus on engineering outcomes

Do not invent achievements.

Do not fabricate metrics.

---

## Phase 8 — Deliverables

Before coding, generate:

1. Portfolio strategy document
2. Information architecture
3. Wireframe description
4. Visual design rationale
5. Content draft

Wait for approval.

Only after approval:

Generate the complete production-ready codebase.

---

## Phase 9 — Coding Rules

After coding:

Run:

* npm install
* npm run lint
* npm run type-check
* npm run build

Fix all issues automatically.

Do not stop until the project builds successfully.

Continue working autonomously until the application is production-ready.

Generate all files.

Do not generate partial code.

Do not leave TODO placeholders.

Do not leave mock content if CV data exists.

Deliver a deployable application ready for Vercel deployment.
